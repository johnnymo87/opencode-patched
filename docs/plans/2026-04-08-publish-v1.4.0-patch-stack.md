### Task 4: Update `workstation` to Use `v1.4.0-patched`

**Files:**
- Modify: `~/projects/workstation/users/dev/home.base.nix`

**Step 1: Write the failing test**

Use a workstation update checklist:

- `version = "1.4.0"`
- all four platform hashes match the published `v1.4.0-patched` artifacts

**Step 2: Run checklist to verify it fails**

Read current file and confirm it still pins `1.3.13`.

**Step 3: Write minimal implementation**

Compute Nix SRI hashes for each artifact:

```bash
BASE="https://github.com/johnnymo87/opencode-patched/releases/download/v1.4.0-patched"
for artifact in opencode-linux-arm64.tar.gz opencode-linux-x64.tar.gz opencode-darwin-arm64.zip opencode-darwin-x64.zip; do
  curl -sL "$BASE/$artifact" -o "/tmp/$artifact"
  echo "$artifact: $(nix hash file --sri --type sha256 /tmp/$artifact)"
done
```

Then update `users/dev/home.base.nix`:
- Change `version = "1.3.13"` to `version = "1.4.0"`
- Replace all four `hash = "sha256-..."` values with the computed hashes
- The hash order in the file is: `aarch64-linux`, `x86_64-linux`, `aarch64-darwin`, `x86_64-darwin`
- These map to artifacts: `linux-arm64`, `linux-x64`, `darwin-arm64`, `darwin-x64`

**Step 4: Run test to verify it passes**

Re-read the file and confirm the version and hashes are updated.

If there is a repo-specific evaluation command available, run it too.
